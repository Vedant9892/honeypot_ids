# Copilot Context: Multi-Service Honeypot IDS Project

This repository implements a cybersecurity research project titled:

**Multi-Service Honeypot Based Intrusion Detection System using Deep Learning and Explainable AI (XAI)**

The goal is to simulate attacker interactions across multiple network services and detect malicious behavior using machine learning.

---

# Project Architecture

The system consists of the following layers:

1. Honeypot Layer (Attack Capture)
2. Logging & Data Collection Layer
3. Feature Extraction Layer
4. Deep Learning Intrusion Detection System (IDS)
5. Explainable AI Layer (XAI)
6. Monitoring Dashboard

The honeypot environment simulates multiple services commonly targeted by attackers.

---

# Simulated Services

The honeypot will simulate the following services:

SSH Service (Port 22)  
FTP Service (Port 21)  
HTTP Web Server (Port 80)

Each service logs attacker behavior such as authentication attempts, commands, and payloads.

---

# Attacks Detected

The system will detect the following attack types:

SSH brute force attacks  
FTP brute force attacks  
SQL Injection attacks  
Cross-Site Scripting (XSS) attacks  
Directory scanning attacks  
Ransomware file encryption behavior

---

# Honeypot Components

## SSH Honeypot

Uses the Cowrie honeypot to emulate an SSH server.

Captured data includes:

- Username attempts
- Password attempts
- Login success/failure
- Commands executed by attacker
- File downloads or malware attempts
- Attacker IP address
- Session timestamps

---

## FTP Honeypot

Implemented using pyftpdlib.

Logs include:

- Username attempts
- Password attempts
- Login success/failure
- FTP commands issued
- File access behavior
- Client IP address

---

## HTTP Web Honeypot

Implemented using a Flask web server with intentionally vulnerable endpoints.

Example endpoints:

/login  
/search  
/upload  
/admin  

Captured web attacks include:

SQL injection payloads  
XSS payloads  
Directory scanning attempts  
Request frequency anomalies

---

## Ransomware Detection Module

The system creates a directory of **decoy files** that simulate valuable documents.

Examples:

salary_data.xlsx  
employee_records.pdf  
client_database.csv  

A monitoring script watches the directory and detects ransomware indicators such as:

- Rapid file modifications
- Mass file access
- File extension changes
- File encryption behavior

---

# Logging System

All honeypot services send structured logs to a central logging module.

Logs are stored in:


data/raw_logs/

Log Format Example:

{
"timestamp": "2026-03-10T12:00:00",
"source_ip": "192.168.56.101",
"service": "ssh",
"event_type": "login_attempt",
"username": "root",
"password": "admin123",
"status": "failed"
}


Logs must be JSON structured for machine learning processing.

---

# Feature Extraction

Raw logs are converted into structured numerical features.

Examples of extracted features:

login_attempt_rate  
failed_login_ratio  
command_frequency  
payload_length  
special_character_count  
request_rate  
file_modification_rate  

These features will be used for training the IDS model.

---

# Deep Learning IDS

The IDS will classify network behavior into categories:

Normal activity  
Brute-force attack  
Web attack  
Ransomware activity

Possible model architectures:

LSTM for sequential behavior  
CNN for feature pattern recognition  
Transformer-based anomaly detection

---

# Explainable AI (XAI)

Explainable AI techniques will be used to interpret model predictions.

Libraries used:

SHAP  
LIME

Example explanation:

"Traffic classified as SQL injection because payload contains SQL keywords and abnormal request frequency."
